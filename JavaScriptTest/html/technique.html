技術面について

　「生産性1.0からのざっくり見積り」を作成するにあたって、いくつかの実験的な試みをしました。 いろいろな事情で仕事ではつかえないツールやライブラリ、興味がある技術などを盛り込んだ集大成です。 初めてのものが多く、調べながら、つまずきながら作ることになり、新たな挑戦は大変なものだなとしみじみ。 とはいえ、Web 畑の人にとってはあたりまえにつかうものなのでしょう。
　ここでは今回の「初めての試み」をリストにしました。

Ubuntu 12.04 LTS 　今までは Windows での開発がメインでしたが、今回はすべて Ubuntu 上で作成しました。 環境は Eclipse 3.7.2 + Tomcat v7.0 とし、ほかにも仕様書の作成には LibreOffice、設計に astah Community など、 Linux で使えるもののみとなっています。
　LibreOffice は Microsoft Office と違う独自な操作があって使いにくかったものの、 とりあえずひと通りのプロセスを Ubuntu だけでこなすのは成功しました。
Struts2 　少し前に携わったプロジェクトでは Struts2 を使用して Web アプリケーションを開発するということだったので、 勉強のためにつかってみようと決めました。が、そのプロジェクトからは実装工程前に撤収することになったため 結局はつかわずじまいに・・・。
　日本語の資料があまりなくて調べるのに苦労しました。
google-gson 　工程工数グラフは JSON 形式のデータをもとに描画してるため、Java で生成したデータオブジェクトを JSON 形式に変換する必要があり、手軽に使えそうなこれを選択しました。
　データ構造がそこそこ複雑になったので、変換は大変かなと心配しましたが、使い方がシンプルでスグにつかえました。
HTML5 　「HTML5 が流行りらしい」という理由でつかってみようとしたものの、結局 HTML5 が何なのか不明。 とりあえず canvas とボックスレイアウトがそれっぽいのかと理解しました。
　工程工数グラフで canvas を初めてつかいましたが、けっこうよさげにできました。 ただ、文字の描画が荒いことや細い線が使えない点が残念です。
JUnit4 　Java 開発ではおなじみのユニットテストツールですが、4 ははじめてつかいました。 ただ調べる手間を惜しんだため、あらたな機能としては assertThat くらいしかつかってません。
　JUnit Helper や Quick JUnit とあわせてつかったところ、とても便利でした。 「やっぱりユニットテストは必要だ、いきなり結合しようとするやつって何なの？」と思うくらい。
jquery 　みんな大好き jquery。いまでは大流行りなのにつかう機会がありませんでした。 セレクタやイベントハンドラがおどろくほどつかいやすくなっていてストレスなく作れました。JavaScript 部分は。
jquery UI 　HTMLのプリミティブなコントロール部品以外をつかう場合に限らず、ボタンなども見栄えがよくなるので使用しました。 ここではスライダやスピナ、ボタンでつかっています。双方向スライダをサポートしてて助かりました。 とくに UI への執着がなければ jquery UI だけで間に合いそうです。
Jquery Canvas Chart 　ポーラーチャートを表示するために使用しました。「俺WEB」氏作のライブラリです。
指定できるパラメータがおおく、こまかい調整が簡単にできてつかいやすかったです。
http://oreweb.toypark.in/jquery.canvasChart/
Uniform 　HTMLのプリミティブなコントロール部品の見栄えをよくしてくれるライブラリです。
ラジオボタンやリストボックスが味気なかったので使用しました。が、あまりうまくつかえてないですね。
http://uniformjs.com/
GIMP 　タイトルロゴをつくるために使用しました。オープンソースでありながら高度な機能があって重宝します。 ただ自分がペイントソフトくらいしかつかってこなかったので、機能に振りまわされてつかいこなせませんでした。
　このソフトだけで色々できそう、だけど自身のデザインセンスのなさが絶望的・・。
GitHub 　Git は CVS と SVN を嫌悪しているという某L氏が開発したということで、興味本位でつかうことを決定。 Linux のような大規模なプログラムを扱うわけじゃなく、ひとりでつかうだけなので用途に合ってないが・・・。
　ろくに調べずにつかっていたのでリポジトリを壊しそうになったり、古いソースでローカルデータを上書き してしまったり、冷や汗をかくことも何度か・・・。
　SVN などより多機能なので、使うときには要注意。
アイコン 　はじめはアイコンも自作しようと考えていたものの、デザインセンスのなさを受け入れて、 素材サイトで提供されているものをつかうことにしました。
　「ICON HOI HOI」さんのアイコンは種類も多く、色ちがい、サイズちがいバリエーションもあり、使いやすかったです。 絵柄も適度にソフトでワンポイントにつかうにはもってこいです。
http://iconhoihoi.oops.jp/index.html
背景画像 　あたたかみがある印象が出せるように、ナチュラル系素材をさがしていて、最終的に木目調におさまりました。
http://subtlepatterns.com/
タイトルロゴ画像 　やわらかめな画像にするか、数学的なものにするか悩んだ末、背景にあわせたやわらか系のタイトルロゴとしました。
　元画像は「YannGar Photo in Lausanne」によって提供されたもので、それを GIMP で編集しています。 GIMP のつかい方がわからなくて大いに苦労しました。
http://free-images.gatag.net/2012/04/09/000000.html
多重スクロール 　某ゲームの 13 多重スクロールや 0.5 ドットスクロールなどの変わったスクロールが好きだったので、 たまたま見つけた sakusaku のサイトを参考にしてみました。
　FireFox だとサクサクとスクロールするのに、 Chromium だとなぜかカクカク・・・。
